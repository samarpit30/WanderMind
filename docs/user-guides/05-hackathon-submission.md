# Hackathon Submission Checklist

Use this guide to ensure all submission details are ready for PromptWars / Build with AI evaluation.

## Submission Checklists

- [ ] **GitHub Repository URL**
  - Verify the repository visibility is set to **Public** (Settings > Danger Zone > Change repository visibility). If it is private, the judges will not be able to evaluate your code.
  - Double check that your final commits (including clean code, docs, and configurations) are fully pushed to the `main` branch on GitHub.

- [ ] **Live Vercel URL**
  - Open the Vercel link in an Incognito window to confirm the page loads without authentication errors or environment key issues.
  - Verify that both the Chat Panel and Map Canvas load successfully.

- [ ] **API Keys & Usage Quota**
  - Ensure your Google Maps billing and API configurations are active so they do not fail when judges test the application during grading.
  - Ensure the Gemini key has remaining free usage tier quota.

- [ ] **Submission Form**
  - Paste your **GitHub Repo URL** in the repository submission box.
  - Paste your **Vercel Live URL** in the deployment link box.
  - Submit the form before the hackathon clock reaches zero!
